# Global Rules

## Language

- Conversations with the user are in Japanese
- All code, comments, commit messages, and documentation must be written in English

## Coding Style

- Keep code as simple as possible
- Write tests whenever feasible

## Commit Messages

- Use Conventional Commits format with a scope: `feat(scope): description`
- Split commits into the smallest semantically meaningful units
- Never commit without explicit user approval

## Workflow

1. **Discuss first**: List relevant facts and options, then agree on a spec with the user before writing code
2. **Implement**: Write the code based on the agreed spec (adjustments during implementation are fine)
3. **Wait for verification**: The user will test and verify the changes
4. **Commit only after approval**: Only create commits once the user confirms the changes work
